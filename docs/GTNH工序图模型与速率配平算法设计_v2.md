# GTNH 工序图模型与速率配平算法设计 v2

## 1. 文档目的

本文档是 `GTNH工序图模型与速率配平算法设计.md` 的第二版设计草案，用于统一后续集成工厂的工序图语义、目标产物语义、环路约束和速率配平规则。

第二版的核心变化是：

```text
结束节点不再表示拓扑终点
结束节点改名为目标产物节点
工序图允许多源头、多目标、多副产链和多个局部环
每个环只允许一种循环物料
循环物料必须具备正净输出
图语义、配平语义和运行模型分开描述
```

本文档只定义工序图模型与配平算法，不直接规定机器状态机细节。机器运行、输入输出、缓存、水位、跳电和换图行为由机器设计文档描述。

---

## 2. 工序图总模型

一张工序图表示一个局部生产网络，而不是全局配方网络。

形式化表示为：

```text
ProcessGraph = (V, E, M)
```

其中：

```text
V = 配方节点集合
E = 物料流有向边集合
M = 图级元信息
```

图级元信息包括：

```text
目标产物节点集合
启动条件需求
节点参数
物料路由推断结果
环路分析结果
配平结果
```

工序图可以包含：

```text
多个无前置节点的源头工序
多个目标产物节点
多个副产物处理链
多个互不相关或局部相连的小环
回收节点等特殊节点
```

工序图中的边只表示物料流关系，不表示 UI 排列顺序，也不直接表示运行调度顺序。

---

## 3. 节点模型

每个节点表示一个虚拟配方运行单元。

节点至少包含：

```text
ProcessNode {
    id
    type
    recipeIdentity
    inputs
    outputs
    baseDuration
    baseEuPerTick
    overclockCount
    parallelLimit
    targetProductNode
    nonConsumableItems
    requiredController
}
```

字段含义：

```text
id = 节点唯一 ID
type = 普通配方、回收节点、虚拟配方等节点类型
recipeIdentity = 配方身份或配方快照
inputs = 配方消耗物料
outputs = 配方产出物料
baseDuration = 节点基础耗时
baseEuPerTick = 节点基础 EU/t
overclockCount = 节点自身设置的超频次数
parallelLimit = 节点基础并行上限
targetProductNode = 是否为目标产物节点
nonConsumableItems = 节点运行资格所需 NC 物品
requiredController = 节点运行资格所需主机方块
```

节点自身参数只描述图语义。机器运行时可以叠加机器级并行倍率和全局额外超频，但这些运行参数不写回节点。

---

## 4. 目标产物节点

### 4.1 命名调整

旧模型中的“结束节点”在 v2 中改名为：

```text
目标产物节点
```

目标产物节点不是拓扑终点。它表示：

```text
该节点的部分或全部产物是本工序图的目标产物
```

目标产物节点后续仍然可以连接其他节点。常见用途是继续处理副产物、废液、回收物或副链原料。

### 4.2 目标产物推断

为了不改变当前 UI，目标产物仍由节点级标记推断。

推断规则：

```text
如果目标产物节点没有下游节点：
    该节点所有产物均视为目标产物

如果目标产物节点存在下游节点：
    该节点输出中，不被任一下游节点消费的产物视为目标产物
    被下游节点消费的产物视为中间物或副产处理输入
```

示例：

```text
目标产物节点 A 输出：
    极不稳定硅岩粉
    酸性硅岩溶液

A -> B，B 消耗酸性硅岩溶液

则：
    极不稳定硅岩粉 = 目标产物
    酸性硅岩溶液 = 后续副产处理链输入
```

### 4.3 目标产物节点校验

提交图时应校验：

```text
至少存在一个目标产物节点
每个目标产物节点至少能推断出一种目标产物
如果目标产物节点所有输出都被下游消费，应提示或拒绝
```

## 5. 物料模型

物料统一抽象为：

```text
MaterialAmount {
    materialKey
    type
    amount
}
```

其中：

```text
type = ITEM 或 FLUID
amount = 物品数量或流体 L 数
materialKey = 稳定物料标识
```

配平算法只比较物料身份和数量，不处理机器库存、ME 输入、输出仓空间等运行期状态。

---

## 6. 边模型

边表示直接物料流：

```text
MaterialEdge {
    fromNodeId
    toNodeId
    materialKey
    type
}
```

含义：

```text
fromNode 的某种产物被 toNode 作为输入消耗
```

边必须满足直接相邻规则：

```text
一条边只连接直接生产者和直接消费者
不表达“全图中某物料最终会被某节点使用”
不表达运行优先级
```

同一种物料可以从一个节点分配给多个下游节点，但配平算法必须处理分流比例。

---

## 7. 物料分类

根据边和目标产物推断，节点产物可分为：

```text
目标产物
中间产物
副产物
循环物料
环净输出物料
```

### 7.1 目标产物

由目标产物节点推断得到，是本图希望获得的主要产物。

### 7.2 中间产物

被后续节点通过边消费的物料。

### 7.3 副产物

不属于目标产物，也没有被后续节点消费的物料。副产物默认外排。

### 7.4 循环物料

在同一个环结构中同时被生产和消费，并参与形成闭环依赖的唯一物料。

### 7.5 环净输出物料

循环物料在环内的总产出大于总消耗时，其差值为净输出。净输出可以作为目标产物、副产物或环外下游输入。

---

## 8. 图结构模型

### 8.1 多源头

无入边节点是源头节点。一张图可以有多个源头节点。

源头节点通常消耗外部输入或启动缓存。

### 8.2 多目标

一张图可以有多个目标产物节点。目标产物节点不是拓扑终点，后续可以继续连接副产链。

### 8.3 多拓扑终点

无出边节点是拓扑终点。拓扑终点不一定是目标产物节点。

### 8.4 副产处理链

目标产物节点之后继续连接的节点通常表示副产处理链。副产处理链的产物可以继续成为目标产物、副产物或回收产物。

---

## 9. 环模型

### 9.1 环的识别

使用强连通分量 SCC 识别环。

满足以下任一条件的 SCC 视为环：

```text
SCC 中节点数量 > 1
单节点存在自环
```

### 9.2 单循环物料约束

每个环只允许一种循环物料。

如果一个环中存在多种物料同时参与回流，则图检查失败。

理由：

```text
多物料循环需要多变量配平
GTNH 配方常带副产、概率和特殊输入，实际很难稳定配平
多物料环会显著增加运行器调度与水位控制复杂度
```

### 9.3 自增环约束

环必须是自增环。

设某环的唯一循环物料为 `C`：

```text
cycleProduced(C) = 环内节点总产出 C
cycleConsumed(C) = 环内节点总消耗 C
cycleNet(C) = cycleProduced(C) - cycleConsumed(C)
```

必须满足：

```text
cycleNet(C) > 0
```

如果 `cycleNet(C) <= 0`，该环不能作为有效工序环。

### 9.4 环启动条件

环必须能够启动。允许的启动来源包括：

```text
启动物料提供循环物料种子
外部输入提供循环物料种子
环内存在不依赖循环物料即可运行的节点
```

如果环完全依赖自身循环物料且没有种子来源，则图检查失败或要求补充启动物料。

### 9.5 环出口规则

循环物料净输出的去向按以下规则推断：

```text
如果循环物料被环外下游节点消费：
    净输出优先作为环外下游输入

如果循环物料是目标产物：
    无下游消费的净输出作为目标产物外排

如果循环物料不是目标产物且没有环外下游消费：
    净输出作为副产物外排
```

其中“循环物料是目标产物”的判定方式是：

```text
环中存在目标产物节点
并且该目标产物节点的目标产物推断结果包含循环物料
```

### 9.6 非循环产物

环内产生的非循环产物仍按普通物料规则处理：

```text
被下游消费 = 中间产物
被目标产物节点推断为目标 = 目标产物
无人消费且非目标 = 副产物
```

## 10. 配平模型

### 10.1 节点速率

节点 `i` 的物料生产速率：

```text
produceRate(i, m) = outputAmount(i, m) × P_i / duration_i
```

节点 `i` 的物料消耗速率：

```text
consumeRate(i, m) = inputAmount(i, m) × P_i / duration_i
```

其中：

```text
P_i = 节点配平并行数
duration_i = 节点固定耗时
```

### 10.2 普通边约束

对于普通 DAG 边，内部物料应满足：

```text
上游分配到该边的生产速率 = 下游通过该边的消耗速率
```

如果一个产物分流到多个下游，则：

```text
上游总生产速率 >= 所有下游消耗速率之和
```

剩余部分按目标产物或副产物计算。

### 10.3 环配平约束

对于每个环 SCC，仅对唯一循环物料建立环约束。

要求：

```text
cycleNet(C) > 0
```

环对外可以压缩为一个虚拟生产单元：

```text
消耗环外输入
维持循环物料内部流转
产出 cycleNet(C)
产出其他非循环物料
```

如果环外下游消费循环物料，则下游只能消费净输出部分，而不能消费维持环运行所需的循环水位。

---

## 11. 目标函数

配平目标是让图稳定生产目标产物，同时尽量减少无意义副产与内部积压。

推荐目标优先级：

```text
1. 所有内部边满足物料守恒
2. 所有环满足单循环物料正净输出
3. 目标产物速率为正
4. 副产物速率可计算并可输出
5. 节点并行数为正整数且不超过限制
6. 在多自由度情况下，优先减少非目标副产与内部残差
```

---

## 12. 图检查规则

提交图时应执行以下检查：

```text
所有节点配方有效
所有边连接物料匹配
所有目标产物节点能推断出目标产物
所有普通内部物料有直接上游来源
所有环 SCC 满足单循环物料规则
所有循环物料具备正净输出
所有环具备启动条件
回收节点输入必须来自直接上游
主机和 NC 需求可统计
```

图检查不处理：

```text
ME 系统是否有足够原料
输出舱室是否有空间
机器运行期能量是否充足
当前内部缓存中是否存在旧图物料
```

这些属于机器运行模型。

---

## 13. 调度顺序建议

配平算法不直接运行图，但应向运行器提供稳定的结构顺序。

推荐顺序：

```text
DAG 部分按上游深度从小到大排序
同深度节点按 nodeId 稳定排序
环 SCC 作为一个局部组处理
环内节点保持稳定顺序
目标产物节点标记不参与拓扑排序
```

上游深度由边决定，不由目标产物节点决定。

---

## 14. 运行模型边界

以下内容只在本文件中给出算法约束，具体执行由机器文档描述：

```text
运行期水位
循环物料 reserve
输出缓存
跳电行为
换图继承 NC 和主机
螺丝刀强制输出
排空模式
```

特别地，旧图运行中产生的中间物、原料缓存、副产物和终产物不参与新图语义比较。新图只比较启动条件，即主机方块和 NC 物品。

---

## 15. 总结

v2 工序图模型将工序图定义为一个允许局部环路的生产网络。

核心规则是：

```text
目标产物节点不是拓扑终点
目标产物通过下游消费关系推断
每个环只允许一种循环物料
循环物料必须正净输出
环净输出按下游消费、目标产物或副产物规则路由
图语义和运行状态严格分离
```

该模型为后续实现流水线调度、复杂副产链、多小环工序和更稳定的配平算法提供基础。
